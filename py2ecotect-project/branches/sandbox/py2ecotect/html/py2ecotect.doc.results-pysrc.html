  7      """ 
  8       
  9      The various solar and thermal analysis calculations store their results  
 10      either directly within the zones they are working on, or in a large 2D  
 11      data array. This method allows access to this array. The exact format  
 12      of data within the array depends upon the type of calculation last  
 13      performed.  
 14   
 15      Parameter(s) 
 16      This property takes the following parameters. 
 17       
 18      i  
 19      As defined below.  
 20       
 21      j  
 22      As defined below.  
 23       
 24      Return Value(s) 
 25      Getting this property returns the following value(s). 
 26       
 27      value  
 28      A decimal value containing the specified array value.  
 29       
 30      Relevant Data Table(s) 
 31       
 32      Thermal Analysis 
 33       
 34      Hourly Temperatures 
 35      Internal temperature values are stored in each zone. Use the following  
 36      command to access them: 
 37       
 38      get.zone.temperature zone hour 
 39       
 40      Hourly Gains  
 41      The seven different gains are stored in the first seven elements of the  
 42      array. Thus the i value refers to the table below whereas the j value  
 43      is the hour index of the day (0 to 23).  
 44       
 45      Data Type Value  
 46      HVAC Load 0  
 47      Conduction Gain 1  
 48      Sol-Air Gains 2  
 49      Direct Solar Gains 3  
 50      Ventilation Gains 4  
 51      Internal Gains 5  
 52      Inter-Zonal Gains 6  
 53       
 54      Heat / Cool Load 
 55      Monthly heating and cooling loads are stored directly in the heating  
 56      and cooling load arrays for each zone. The 'Outside' zone (index = 0)  
 57      stores the total loads of all zones visible at the time of the last  
 58      calculation. Use the following commands to access them: 
 59       
 60      get.zone.heating zone month 
 61      get.zone.cooling zone month   
 62       
 63      Temperature Distribution (0 - 48) 
 64      This too is stored within the heating and cooling load arrays for each  
 65      zone. As there are only 12 indexes in each load array (1 for each  
 66      month), each index represents a 2-degree range with the heating array  
 67      representing 0-24 deg Cel and the cooling array 24-48deg Cel. 
 68       
 69      Solar Exposure 
 70      Single Day 
 71      The data from a single day's solar radiation analysis on a surface is  
 72      not stored in the results array. Use the get.results.solar command  
 73      described below. 
 74       
 75      Average Daily & Total Monthly 
 76      These values are stored in the array by MONTHS and HOURS. The i value  
 77      refers to the month (0-11) whereas the j value is the hour index of the  
 78      day (0 to 23). The array only stores the current radiation data value.  
 79      Thus, if the Radiation Data: Item is set to Absorbed, then only the  
 80      absorbed value is stored and displayed.  
 81       
 82      An annual value is also stored for i=12, but not displayed. For Total  
 83      Monthly calculations, these are the annual totals for each hour. For  
 84      Average Daily calculations these are the annual average daily value  
 85      for eah hour. 
 86       
 87      Full Hourly 
 88      This calculation stores all 8760 hourly values in the array by DAY and  
 89      HOUR. The i value refers to the day of the year (0-364) whereas the j  
 90      value is the hour index of the day (0 to 23).  
 91       
 92      Resource Use 
 93      The resource use values are stored in the array by day and type. The i  
 94      value refers to the day of the year (0-364) whereas the j value is 0  
 95      for the resource used or 1 for the resource collected. 
 96       
 97      """ 
 98      arg_str = p2e._util._convert_args_to_string("get.results.array", i, j) 
 99      val = p2e._app.Request(arg_str) 
100      return p2e._util._convert_str_to_type(val, float)
